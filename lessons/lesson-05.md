# Заняття 5 — Валідація

## Валідація даних на стороні сервера

- Навіщо потрібна валідація — захист від некоректних даних, безпека, цілісність бази даних
- Типи валідації: вхідні дані (`req.body`), параметри маршруту (`req.params`), параметри запиту (`req.query`)
- Проблеми без валідації — некоректні дані в базі, падіння сервера, вразливості безпеки

## Бібліотека `celebrate`

- Флоу валідації:
  - Маршрут → middleware валідації → контролер
  - Маршрут → middleware валідації → middleware обробки помилок валідації
  - Якщо валідація успішна — запит іде до контролера
  - Якщо валідація не пройшла — помилка передається до middleware обробки помилок
- Middleware `errors()` з `celebrate` для обробки помилок валідації
- Підключення middleware валідації у маршрутах
- Організація схем валідації у `src/validations/`
- Схеми валідації з `Joi`
- `Segments` — валідація різних частин запиту:
  - `Segments.BODY` — тіло запиту
  - `Segments.PARAMS` — параметри маршруту
  - `Segments.QUERY` — параметри запиту

## Валідація ідентифікатора MongoDB

- `isValidObjectId` з Mongoose — перевірка чи є рядок валідним ObjectId
- Валідатор `custom` у Joi — підключення власної логіки валідації

## Послідовність заняття

1. Пояснити навіщо потрібна валідація — показати що відбувається без неї
2. Встановити `celebrate` та розібрати флоу валідації
3. Створити першу схему валідації з `Joi` для `POST /students`
4. Підключити middleware валідації до маршруту
5. Підключити `errors()` middleware для обробки помилок валідації
6. Організувати схеми у папку `src/validations/`
7. Додати валідацію `req.params` через `Segments.PARAMS`
8. Додати валідацію `req.query` через `Segments.QUERY`
9. Реалізувати валідацію MongoDB ObjectId через `isValidObjectId` та `custom` валідатор
10. Протестувати валідацію через Postman — показати `400` відповіді
