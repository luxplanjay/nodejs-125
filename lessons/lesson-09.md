# Заняття 9 — Скидання паролю та електронна пошта

## JWT (JSON Web Token)

- Що таке JWT і навіщо він потрібен
- З чого складається JWT:
  - `header` — алгоритм підпису
  - `payload` — дані: `sub`, `name`, `iat`
  - `signature` — підпис для перевірки автентичності
- `sub` — унікальний ідентифікатор користувача
- `iat` — час видачі токена
- Генерація токена через `jsonwebtoken` — `jwt.sign()`
- Верифікація токена через `jsonwebtoken` — `jwt.verify()`

## Скидання паролю

Типовий флоу скидання паролю:

1. Користувач вводить email
2. Сервер генерує JWT токен і надсилає посилання на email
3. Користувач переходить за посиланням з токеном
4. Сервер перевіряє токен і дозволяє встановити новий пароль

## Відправка email через Nodemailer

- [`nodemailer`](https://www.npmjs.com/package/nodemailer) — бібліотека для відправки email
- [Brevo](https://www.brevo.com/) — SMTP сервіс для відправки листів
- Налаштування транспорту через SMTP credentials з Brevo
- Змінні оточення для SMTP — `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASSWORD`

## Реалізація в додатку

`POST /auth/request-reset-email` — запит на скидання паролю

- Прийняти `email` з тіла запиту
- Знайти користувача за `email` — повернути `404` якщо не знайдено
- Згенерувати JWT токен з `userId` та `email` через `jwt.sign()`
- Надіслати лист з посиланням для скидання паролю через `nodemailer` + Brevo

`POST /auth/reset-password` — встановити новий пароль

- Прийняти `token` та `password` з тіла запиту
- Верифікувати JWT токен через `jwt.verify()` — повернути `401` якщо протермінований
- Знайти користувача за даними з токена
- Хешувати новий пароль та зберегти в базі даних
- Видалити всі активні сесії користувача

## Послідовність заняття

1. Пояснити структуру JWT — header, payload, signature
2. Показати `jwt.sign()` та `jwt.verify()` на простому прикладі
3. Зареєструватись у Brevo — отримати SMTP credentials
4. Налаштувати `nodemailer` з Brevo через змінні оточення
5. Реалізувати `POST /auth/request-reset-email` — генерація токена та відправка листа
6. Реалізувати `POST /auth/reset-password` — верифікація токена, оновлення паролю, видалення сесій
7. Протестувати через Postman — перевірити флоу скидання паролю
