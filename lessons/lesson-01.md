# Заняття 1 — Вступ до Node.js

## Node.js

- Що таке Node.js і чим відрізняється від браузерного JS
- Виконання JS поза браузером через `node index.js`
- Ініціалізація проєкту з `npm init`
  - Властивість `type: module` — підтримка ESM
  - `nodemon` — автоматичний перезапуск сервера при змінах
- Глобальні об'єкти
  - `process.env` — змінні оточення
  - `import.meta.url` — шлях до поточного файлу
- Вбудовані модулі Node.js
  - `node:path` — побудова кросплатформних шляхів до файлів
  - `node:fs` — читання та запис файлів

## Практика з файловою системою

- Прочитати `assets/cars.json` через `fs.promises.readFile`
- Відфільтрувати автомобілі за типом або брендом
- Записати результат у новий файл через `fs.promises.writeFile`

## Архітектура клієнт-сервер

- REST API — що це і навіщо
- Клієнт та сервер — хто кому надсилає запити
- HTTP-методи: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`
- Статуси відповідей: `200`, `201`, `400`, `404`, `500`
- Заголовки запиту та відповіді
- Шляхи та ресурси — що означає `/api/cars`
- Параметри запиту (`query params`) — `?brand=BMW&type=SUV`
- Демонстрація через Postman

## Стартові файли у проєкті

- `assets/cars.json` — масив даних автомобілів для демонстрації роботи з файловою системою

## Послідовність заняття

1. Показати що Node.js це JS поза браузером — запустити простий скрипт через `node index.js`
2. Ініціалізувати проєкт через `npm init` — пояснити `type: module` та підключити `nodemon`
3. Розібрати `process.env` та `import.meta.url`
4. Показати `node:path` — побудова шляху до `assets/cars.json`
5. Прочитати `assets/cars.json` через `fs.promises.readFile`
6. Відфільтрувати автомобілі за типом — записати результат у новий файл через `fs.promises.writeFile`
7. Пояснити архітектуру клієнт-сервер — клієнт, сервер, HTTP-методи, статуси, заголовки
8. Показати `query params` на прикладі — `?brand=BMW&type=SUV`
9. Демонстрація через Postman — надіслати GET запит та переглянути відповідь
